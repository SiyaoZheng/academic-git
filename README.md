# Fu

> “学术则黜伪而崇真，于刑政则屈私以为公。”
>
> 严复，《论世变之亟》

本项目名为 Fu。项目名一部分来自严复之“复”，借其以西方科学方法重释中国思想世界之意；另一部分来自 `FOO/BAR（程序示例中的占位符）`。前者对应思想史与科学方法，后者对应程序书写中的占位符。这个命名同时指向 AI（artificial intelligence，人工智能）支持的社会科学研究的基础工作。

Fu 面向计算社会科学、计算政治学，以及更广义的人文社会科学研究者。许多社会科学研究者已经能够使用 Python（编程语言）或 R（统计语言）完成数据清洗、文本分析、统计建模和可视化工作。

计算能力的扩展带来了新的便利，也把新的责任放到了研究过程上。代码、数据、版本、结果和写作之间只要对应关系出现问题，昨天能够运行的回归在同样条件下就会失去复现性，图表进入论文后生成脚本的来源就会变得难以说明，假设在结果出现之后被重新组织，合作者看到定稿，也能够看到中间的判断、删改与取舍。

这些问题直接关系到社会科学研究如何处理证据、如何约束解释、如何区分事前设定与事后修正。严复所谓“黜伪而崇真”，放在今天的计算社会科学语境中，意味着研究过程应当同时留下结论和过程记录。Fu 要做的，就是把这一要求落实到日常工作流中。

## 适用对象

本项目主要面向刚开始使用 vibe coding（氛围式编程）的人，也面向希望在研究与写作中保留可追溯性、可复核性与过程记录的人文社科团队和项目负责人。

如果目标是建立一套面向 AI（artificial intelligence，人工智能）支持的社会科学研究基础设施，这些规整直接服务于研究结论的可信度。

## 使用方式

开始一个新的研究问题或新的分析时，研究者开启一个 session（会话）。一个 session 只对应一个研究任务，研究任务可以跨多个 session 延续，但边界始终保持清楚。

在该 session 中，研究者照常完成数据清理、文本分析、统计建模、图表整理和写作。Fu 负责记录、整理和保存，研究者只需继续推进当前任务。

## 设计理念

### 基本设想

Fu 以 artifact（研究产物）的可追溯性为中心。代码、数据、图表、文本、回应和结论都应当留下来源、版本和修改痕迹。研究过程的价值，和最终文本一样，需要被保存下来。

### 概念入口

Fu 的基本规则只有一条。一次 session 只对应一个研究任务。任务可以延续，任务边界始终保持清楚。

### 复现、假设与复核

Fu 追求的不是只记录代码和结果的 reproducible social science（可复现社会科学）。代码和结果往往已经接近研究的收束阶段，Fu 更关心的是探索过程本身，也就是 research process（研究过程）和 exploratory process（探索过程）在每一次尝试、修正与判断中的留痕。

研究代码常见的风险来自环境、依赖、路径和中间文件的变化。Fu 关注的是结果在什么条件下生成，以及这些条件是否能够被重新确认。HARKing（hypothesizing after the results are known，结果后设假设）也需要认真处理，分析前的设想与分析后的修正都应当留有记录。

后续若有调整，也应当以追加记录的方式保留下来。这样，最初的问题、后来的修正和最后的结果都会留在同一份记录中。论文中的表格、图像和结论，理想情况下都应当能够追溯到对应的过程记录。这样，合作者、审稿人或未来的自己看到的既有最终文本，也有完整的研究路径。

Fu 关注的中心因此是研究产出能否被定位、复现、解释和复用。换言之，代码只是手段，真正需要被保护的是研究产物及其生成过程。
