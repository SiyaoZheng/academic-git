Architecture
Thick Skill, Thin Harness

Source of Truth
Issue = SSOT
Body = original plan (immutable). Comments = changelog (append-only).
Mapping
1 Issue = 1 Branch = 1 Worktree = 1 PR
Each task: one Issue, one linked codex/issue-<number>-<slug> branch/worktree, one squash-merged PR.

Architecture

Codex hooks + skills. No intermediate layers. GitHub is the database.

Hooks (Stop + SessionStart) hooks/codex/*.sh → wip snapshots + crash recovery + fire-and-forget push
Skills (Codex invokes directly) /begin → /codex-gh-issue-start → /commit ← /refine-issue
GitHub (SSOT) Issue body (immutable plan) + Issue comments (append-only changelog) + Checklist [x] (progress)

Workflow

From session start to PR merge — zero manual git commands.

Formal Commit

          
/commit skill — 1 commit = 1 checklist item

        

      

      

        Read Issue (SSOT) → analyze changes → Codex reviews diff → commit selected paths as type(#N/X): desc → push → check off item [x] only when complete.

        When all items are [x]: generate PR body with Closes #N → create gated PR → squash merge → return to the default branch.

Issue Refinement

          
/refine-issue skill — mid-stream scope changes

        

      

      

        Scope changed? Need new items? Item no longer needed?
        Issue body is never modified. All changes are logged as timestamped, append-only comments.
        Codex reconstructs current truth by reading body + all comments sequentially.
      


      

        
LayerWhat It HoldsMutability

        
          

            Issue body
            Original plan (checklist, scope, context)
            Immutable — frozen at creation
          

          

            Issue comments
            All modifications since creation
            Append-only — never edit, never delete
          

        
      


      

        

          
skills/refine-issue/SKILL.md Comment-only refinement, append-only audit trail

7

        

          
Completion

          
All checklist items [x] — fully automatic

        

      

      

        Codex detects all items checked off → Codex reviews full diff → create PR with Closes #N → squash merge → return to the default branch.

        No "done" signal from Adrian needed. Milestone deliveries get tags: email-YYYY-MM-DD, meeting-YYYY-MM-DD, conference-YYYY-MM-DD.
      

    


  







  

    
Anti-ADHD Safeguards

    
Structural guardrails that don't rely on human discipline.


    

      

        
Session Start Drift

        
/begin forces Issue + item selection via AskUserQuestion before any work

      

      

        
Mid-Task Tangent

        
Mandatory Issue + item parameter on every /commit — can't commit untracked work

      

      

        
Scope Creep

        
DAG enforces item ordering — only unblocked items startable

      

      

        
Forgot to Save

        
hooks/codex/stop.sh snapshots after every Codex response

      

      

        
Crashed / Closed Laptop

        
hooks/codex/session-start.sh restores branch and issue context on resume

      

      

        
Never Says "Done"

        
Codex checks: all checklist [x] = ready for PR + merge
