# Project on Social Cleavages in China: transition case

## 这是什么 case

这个项目不是一开始就“干净、稳定、完全规范”的模板。它同样有混乱的起点。但后续逐步出现了另一种形状：issue、branch、worktree、测试和更清楚的任务边界开始形成治理结构。

## 为什么它重要

它最适合被写成一个 transition case，而不是完美案例。

- 起点说明研究项目天然会滑向混乱
- 过渡说明一旦出现治理层，history 的形状会发生变化
- 结果说明研究工作可以逐步变得可审计、可分段、可回看

## ScholarOS 从这个 case 里看到什么

- 治理层不是装饰，它会改变研究历史的形状
- 过程治理并不要求研究一开始就完美
- 真正重要的是从混乱走向可解释，而不是假装从未混乱

## 对应的 solo research Git flow 转变

这个 case 的价值在于，它显示个人研究可以从混乱进入分层结构：研究问题进入明确任务边界，任务边界进入局部研究单元，局部研究单元再沉淀为可解释的 artifact。ScholarOS 把它固定为 transition case，是为了避免把真实研究写成一开始就完美的流程。

## 这个 case 之后要补什么

- 更具体的 before / after evidence
- 更清楚的任务边界样例
- 与 failure case 的对照表
