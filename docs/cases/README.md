# Case Archive

ScholarOS 的 case archive 不把 case 当作 marketing demo，而把它们当作方法论证的一部分。

## 当前 case 对

1. [scientificity: failure case](scientificity-failure-case.md)
   没有稳定治理层时，研究会继续发生，但过程会逐步消失。

2. [Project on Social Cleavages in China: transition case](social-cleavages-transition-case.md)
   研究项目天然会滑向混乱，但当 issue、branch、worktree 和更清楚的任务边界开始出现时，history 的形状会发生变化。

## 这个 archive 的作用

- 证明 ScholarOS 关心的不是抽象规范，而是真实研究现场里的问题
- 让读者先看到 evidence，再接受概念
- 让更广泛的社会科学同行理解，这不是给程序员看的工作流玩具

## 写作规则

- case 先写研究问题和过程风险，再写技术承接
- 不把 hooks、skills、GitHub 术语放在第一段
- 保持 failure / transition / later success 的区分，不把真实工作抹平成完美流程
