# Question -> Session -> Artifact -> Paper

ScholarOS 当前最核心的一张图，不是技术架构图，而是研究过程图。它对应的是 ScholarOS 的过程记忆结构：研究从问题出发，在会话中形成边界，在 artifact 中留下来源，最后才进入论文、报告、讲稿或图表。

```text
研究问题
   |
   v
会话 / session
   |
   v
探索、试错、修正、冻结
   |
   v
研究产物 / artifact
   |
   v
论文、报告、讲稿、图表
```

## 这张图在说明什么

ScholarOS 的中心不是“自动生成更多东西”，而是把最终产物重新接回生成过程。

- `question`
  说明研究从什么问题出发。

- `session`
  说明工作在哪个任务边界内展开。

- `artifact`
  说明代码、图表、文本、数据处理与判断如何留下来源。

- `paper`
  说明最终公开文本并不是凭空出现，而是从一条可回看的过程链中长出来。

## 为什么它重要

如果缺少中间这两层，研究者最后通常只剩下两个孤立对象：

- 一个已经成稿的论文
- 一堆难以解释来源的脚本和中间文件

ScholarOS 要做的，就是让这中间的链条重新变得清楚。
